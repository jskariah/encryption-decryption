# Ciphers Encryption and Decryption Project

## Overview

This project is a C++ command-line application for encrypting and decrypting text using classical cryptography techniques. The program supports both Caesar ciphers and substitution ciphers, along with English-language scoring using quadgram frequency analysis.

The main implementation work for this project was completed in `ciphers.cpp`.

The application allows users to:

- Encrypt text using a Caesar cipher
- Decrypt Caesar ciphers using dictionary matching
- Apply random substitution ciphers
- Decrypt substitution ciphers using hill-climbing optimization and English scoring
- Compute the Englishness score of text using quadgram frequency analysis
- Decrypt substitution ciphers directly from files

## Features

### Caesar Cipher Encryption

Encrypts plaintext by rotating letters through the alphabet by a user-provided shift amount.

### Caesar Cipher Decryption

Attempts to automatically decrypt Caesar ciphers by:

- Rotating through all 26 possible shifts
- Comparing resulting words against a dictionary
- Printing likely decryptions based on English word matches

### Englishness Scoring

Computes an English-language likelihood score using quadgram frequency analysis.

### Random Substitution Cipher Encryption

Generates and applies a random substitution cipher to plaintext.

### Substitution Cipher Decryption

Decrypts substitution ciphers using:

- Randomized key generation
- Hill-climbing optimization
- Quadgram scoring
- Multiple search attempts to improve results

### File-Based Cipher Decryption

Decrypts ciphertext directly from an input file and writes the resulting plaintext to an output file.

## How Substitution Cipher Decryption Works

The substitution cipher decryption system uses a hill-climbing optimization approach.

The algorithm:

1. Generates a random substitution key
2. Applies the key to the ciphertext
3. Scores the resulting plaintext using English quadgram frequencies
4. Randomly swaps letters in the key
5. Keeps changes that improve the Englishness score
6. Repeats the process multiple times to improve accuracy

This allows the program to approximate likely plaintext for substitution-encrypted text without knowing the original key.
